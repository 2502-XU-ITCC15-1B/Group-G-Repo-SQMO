# Project Name

## Overview

This project is a web application intended to develop a web-based document access system for the Strategy and Quality
Management Office (SQMO) of Xavier University - Ateneo de Cagayan. The system will serve
as an organized, secure, and user-friendly interface that allows accreditors to log in and
navigate self-survey report documents through a structured dashboard with direct links to
Google Drive folders.

At the current stage, the project focuses on initial front-end implementation and basic version control workflows.

---

## Current Progress

The project has completed the following:

### Step 1: Feature Development

* A basic front-end was created in a dedicated feature branch.
* The front-end includes:

  * Initial UI layout
  * Basic structure for future features
  * Placeholder components

### Step 2: Branch Integration

* The feature branch was successfully merged into the `develop` branch.
* Merge conflicts (if any) were resolved during integration.
* The `develop` branch now contains the latest working version of the front-end.

---

## ⚠️ Incomplete Stages

The following stages have **NOT yet been implemented**:

* Merge from `develop` to `staging`
* Semantic versioning (alpha/beta releases)
* CI/CD pipeline setup (e.g., via GitHub Actions)
* Deployment to hosting platforms such as:
  * Vercel
  * Render
  * Microsoft Azure
* Staging environment setup
* Acceptance testing (alpha and beta phases)
* User feedback collection and documentation

---

## Known Limitations

* The application currently includes only a basic front-end.
* No back-end functionality is implemented.
* No deployment or live environment is available.
* Features are not fully functional and are primarily placeholders.

---

## Next Steps

Planned future work includes:
